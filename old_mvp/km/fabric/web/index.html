How It Works

KnowledgeMesh connects AI buyers and sellers in a decentralized marketplace. No middlemen, no markup — just peer-to-peer inference at a fraction of API prices.

🛒
Buyer
Send inference requests using curl, Python SDK, or any OpenAI-compatible client. Pay only for tokens used.
🌐
Broker
Routes tasks to the cheapest eligible node. Handles escrow, billing, and model verification automatically.
⚙️
Seller
Run the lightweight worker with your API key or subscription. Earn credits when buyers use your node.

Recommended

Register and start your node in one command. No dashboard needed.

For Buyers

Three ways to buy inference from the network.

1. curl (direct API)

curl -s -X POST https://km-broker.onrender.com/task \ -H "Content-Type: application/json" \ -d '{ "buyer": "your-node-name", "buyer_secret": "km-sec-xxx", "messages": [{"role": "user", "content": "What is quantum computing?"}] }' | python3 -m json.tool

2. Python SDK

pip install knowledgemesh from knowledgemesh import KM km = KM(secret="km-sec-xxx") result = km.chat("What is quantum computing?") print(result["content"]) print(f"Cost: ${result['cost']:.6f}, saved {result['savings']:.0f}% vs API")

CLI also available: KM_SECRET="km-sec-xxx" km-chat "Your prompt"

3. OpenAI-compatible endpoint

Drop-in replacement for LangChain, LlamaIndex, Cursor, or any OpenAI-compatible tool. Just change the base URL.

curl -s https://km-broker.onrender.com/v1/chat/completions \ -H "Authorization: Bearer km-sec-xxx" \ -H "Content-Type: application/json" \ -d '{ "model": "claude-sonnet", "messages": [{"role": "user", "content": "Hello!"}] }' | python3 -m json.tool

Python (OpenAI SDK compatible)

from openai import OpenAI client = OpenAI( base_url="https://km-broker.onrender.com/v1", api_key="km-sec-xxx" # your node secret ) response = client.chat.completions.create( model="claude-sonnet", messages=[{"role": "user", "content": "Hello!"}] ) print(response.choices[0].message.content)

How pricing works

  • You set your price per million tokens when you register
  • The cheapest online node gets tasks first
  • Credits are escrowed before each task, settled after completion
  • Failed tasks get full refund — no risk to buyers
  • Change your price anytime via the dashboard or API

Change tier or price via API

# Update price curl -s -X POST https://km-broker.onrender.com/update-price \ -H "Content-Type: application/json" \ -d '{"name":"your-node","node_secret":"km-sec-xxx","price_per_million_tokens":1.00}' # Change tier curl -s -X POST https://km-broker.onrender.com/update-tier \ -H "Content-Type: application/json" \ -d '{"name":"your-node","node_secret":"km-sec-xxx","tier":"openai"}'

Web Search

Some models support web search natively through KnowledgeMesh.

OpenAI — request gpt-4o-search-preview

Web search is automatic when you use this model. No extra parameters needed.

curl -s -X POST https://km-broker.onrender.com/v1/chat/completions \ -H "Authorization: Bearer km-sec-xxx" \ -H "Content-Type: application/json" \ -d '{ "model": "gpt-4o-search-preview", "messages": [{"role": "user", "content": "Latest news on AI"}] }'

Anthropic — add web_search_options

Include "web_search_options":{} in the request body.

curl -s -X POST https://km-broker.onrender.com/v1/chat/completions \ -H "Authorization: Bearer km-sec-xxx" \ -H "Content-Type: application/json" \ -d '{ "model": "claude-sonnet", "messages": [{"role": "user", "content": "Latest news on AI"}], "web_search_options": {} }'

Security

  • Node secrets are generated server-side and shown only once at registration. They authenticate all API calls.
  • Emails are stored as SHA-256 hashes — never in plaintext. Used only for account recovery.
  • Account recovery is available via /recover using your node name + email. A reset token is sent to your email.
  • API keys and session cookies are never sent to the broker. They stay on your local worker only.
  • Tunnel connections use Cloudflare Tunnel for secure, encrypted communication between broker and workers.

Task Request Body

{ "buyer": "your-node-name", "buyer_secret": "km-sec-xxx", "messages": [ {"role": "user", "content": "Your prompt"} ], "model": "claude-sonnet", // optional: request a specific model "node": "preferred-node", // optional: pin to a specific worker by name "tier_preference": "subscription", // optional: api, subscription, ollama, openai "max_budget": 5.0, // optional: max credits to spend "max_tokens": 5000 // optional: max tokens for accurate escrow }
FieldTypeRequiredDescription
buyerstringyesYour registered node name
buyer_secretstringyesYour node secret for authentication
messagesarrayyesChat messages (OpenAI format: role + content)
modelstringnoRequest a specific model (e.g. "claude-sonnet", "gpt-4o"). Supports prefix matching.
nodestringnoPin the task to a specific worker node by name. Useful for session affinity.
tier_preferencestringnoPrefer a specific tier: api, subscription, ollama, openai
max_budgetnumbernoMaximum credits to spend on this task
max_tokensintegernoMax tokens for accurate escrow calculation. Also filters nodes without enough budget.
